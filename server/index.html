Тестовое задание AppFollow для фронтенд разработчика.

Нужно спроектировать страницу с модальным окном для ввода биллинг данных компании.

Сервер с данными лежит здесь. Запускатся с помощью команды `node index.js`.

Тестовое нужно выполнять в отдельном репозитории.

Требования

1. На странице кнопка "Ввести данные", по нажатию на которую открывается модалка. Модалку можно закрыть.

2. В модальном окне должны быть следующие поля с валидацией:

Поле Тип Валидация Заметка
Name Input Минимум 2 слова по 2 буквы Ахтунг! Две буквы, не символа
Company Input Минимум 3 символа
Country Dropdown Страна должна быть из списка Список стран со штатами приходит с /billing_info в поле countries
State Dropdown Штат должен быть из списка соответствующей страны Массив штатов есть не у каждой страны
VAT ID Input Не пустое только у стран из ЕС, валидировать лучше библиотекой Инпут показывается только у стран из ЕС
ZIP Code Input Минимум 3 символа
Address Textarea Минимум 7 символов

2.1 Ошибка валидации должна исчезать после введения в невалидное поле любого символа.

3. Модалка должна инициализироваться с "сохраненными" данными с бекенда, получаемыми с /billing_info.

4. Данные должны отправляться на /save. После получения {ok: 1} с сервера модальное окно закрывается.

5. Написать это все на React (cra или кастомный конфиг), остальные библиотеки по желанию, но без фанатизма.

5.1. Библиотеки для форм, например redux-forms использовать нельзя.

Для экономии времени рекомендуем использовать UI библиотеку, у нас semantic-ui.

Описание API

        GET: /billing_info
        {
            countries: Array<{
                country: string,
                states?: <string>
            }>

            data: {
                company: string,
                country: string,
                vat: string,
                zip_code: string,
                city: string,
                address: string,
                state: string,
            }
        }


        POST: /save
        {
            company: string,
            country: string,
            vat: string,
            zip_code: string,
            city: string,
            address: string,
            state: string,
        }

Что будем проверять:

  1. Чистоту кода.
  2. Разбиение кода на куски (можно ли назвать это архитектурой в таком маленьком приложении?).
  3. Соотвествие требованиям и отсутствие багов.
  4. Бест практисы.
